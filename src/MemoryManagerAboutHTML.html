How to Use

Adding Processes

You can add processes in two ways. The first way is to manually input a process. Click on the 'Add New Process' button. An input box will pop up and ask you for the length of the process. Type in the length and press enter. The second way is to add a random process by clicking the 'Add Random Process' button. The process's length will range from lengths 10 to 10d0, inclusively.

If the memory box has enough space, the process will be placed in memory. How it will be placed depends on the radio box selected.

  • First Fit: The process will be stored in the first hole of memory it can fit in.
  • Best Fit: The process will be stored in the hole of memory that leaves the smallest hole when placed.
  • Worst Fit: The process will be stored in the hole of memory that leaves the largest hole when placed.

For Best Fit and Worst Fit, if there exist two or more holes of the same length that fit the respective criteria, the hole will be chosen via First Fit, i.e. the hole with the smallest base address will be chosen.

If the memory box does not have enough space, the memory will automatically compact in an attempt to make a hole big enough for the process to be placed. If that doesn't work, the process will be placed on a waiting queue. This queue's scheduling algorithm is First In, Last Out, meaning the least recently placed process in the queue will always be the next to be placed in memory.

Deleting Processes

To delete a single process, press the 'Remove Process' button. A dialog box will appear that will ask you to specify which process to remove. Select the desired process and press OK. If any of the processes can fit in the hole that the process was in, a popup box will appear, notifying you that the process is being placed into memory.

To delete all processes, press the 'Reset' button. When pressed, all processes in memory and in the waiting queue will be deleted.

Compacting

It's easy to compact the processes in memory; simply press the 'Compact' button, and the processes in memory will compact. If the hole created via compaction is big enough to place the last process of the waiting queue, that process will automatically be placed into memory.

Assumptions

  1. Up to 15 Processes may be stored cumulatively.
  2. The waiting queue uses a First In Last Out algorithm.
  3. Processes must be between 10 and 250 KB. [inclusive]
  4. The length of memory is at a fixed size of 500 KB.
